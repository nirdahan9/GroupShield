## תמצית מנהלים

מתוך ~250 אירועי אכיפה שנותחו:
- **False Positives (חסם מה שלא צריך):** ~30 מקרים
- **False Negatives (פספס מה שצריך לחסום):** ~20 מקרים
- **תקין:** רוב האכיפה על קללות ישירות עבדה נכון

**הבעיות הכי חמורות:**
1. הLLM העביר תוכן פוגעני קשה ביותר — כולל רמיזות לאונס, תוכן מיני מפורש, הפניה לתאי גז, השוואה לפדופיל
2. מנגנון ה-rule_engine חסם עשרות מילים תמימות בגלל substring matching ללא גבולות מילה

## חלק א — FALSE POSITIVES (חסם בטעות)

### קטגוריה 1: מילים תמימות שמכילות שורש קצר של קללה כ-substring

אלו הבאגים הכי עקביים. מילות קללה קצרות (3–4 אותיות) נמצאות בתוך מילים יומיומיות תמימות:

| הודעה | מה מחסים | מדוע זה שגוי |
|--------|-----------|--------------|
| `חריגה` (×3! כולל "איך ח ר י ג ה זה קללה") | `חרי` בתוך `חריגה` | "חריגה" = exception — מילה נפוצה. אירוני: שאלו למה נחסם ונחסמו שוב |
| `אנטיוכוס` | `כוס` בסוף השם | שם היסטורי |
| `יבור ללא תחתית` | `תחת` בתוך `תחתית` | "תחתית" = bottom, שכיח בשיח אדריכלי/מתמטי |

### קטגוריה 2: LLM לא מזהה Latin-בתוך-עברית (דפוס עקבי)

כל המקרים הבאים עברו — דפוס אחד: **מחליפים אות עברית באות לטינית דומה**:

| הודעה | מה המשמעות האמיתית |
|--------|---------------------|
| `לxנוס` | לאנוס (x = א) |
| `מfגר` | מפגר (f = פ) |
| `מIצץ` | מוצץ (I = ו) |
| `גxי` | גיי כגידוף (x = י) |
| `בuלבuל של אמא` | בילבול/פין (u = י) |
| `מז7ייN` | מזיין (7 = י, N = ן) |

**הLLM מקבל text גולמי, רואה מלים שבורות, ומאשר.** הrule_engine לא שולח אותן ללLM כי אין בהן עברית נקייה ואין תו מיוחד (*@#!).

### קטגוריה 3: גידופים ערביים שלא ברשימה

| הודעה | מה קרה | מדוע זה חמור |
|--------|---------|--------------|
| `יא דבע` (×3 ממשתמשים שונים!) | llm_pass כל פעם | `דבע` בערבית = לסדום — גידוף מיני קשה מאד |
| `ניגר` | לא נחסם, לא נשלח לLLM | ה-n-word בכתיב עברי — חסר מהרשימה לחלוטין |
| `חמאר` | לא נחסם | ערבית: חמור = טיפש — גידוף נפוץ |
| `יכלב` / `יכאלב` | לא נחסם | גרסאות ערביות/היברידיות של "כלב" כגידוף |

### קטגוריה 4: מקרים נוספים שעברו

| הודעה | מה קרה |
|--------|---------|
| `ז*ן` | isSuspicious שלח לLLM, LLM אישר — "זין" עם * לא זוהה |
| `מז7ייN` (ללא "בתחת") | LLM אישר — rule_engine תפס רק בגלל "תחת" |
| `לגזים` / `כולכם לגזים` | לא ברשימה, LLM אישר |

## חלק ד — ניתוח שורשי הבעיות

### בעיה 1: Substring matching ללא גבולות מילה בעברית

**הסבר טכני:** `smartMatchesForbidden` בודק `normMessage.includes(normForbidden)`. לכן כל מילה שמכילה substring של קללה — נחסמת. בעברית אין `\b`, אבל ניתן לבדוק שלפני/אחרי המילה אין אות עברית.

**מילים שגורמות לבעיה הכי הרבה:**
- `כוס` (3 אות) — נמצאת בתוך `אנטיוכוס`, `בכוס` (cup), ועוד
- `חרי` (3 אות) — נמצאת בתוך `חריגה`, `חריג`, `חריף`
- `תחת` (3 אות) — בתוך `תחתית`, `תחתון`
- `נבל` (3 אות) — בתוך `מנבל`, `מנבלים`

**הפתרון:** להוסיף word-boundary עברי לכל מילה ≤ 4 אותיות.

### בעיה 2: LLM לא מזהה Latin-inside-Hebrew bypasses

**הסבר:** `isSuspicious()` מזהה `*`, `@`, `#`, `!` בין אותיות, אבל **לא** אות לטינית בודדת המוחלפת. מישהו שכותב `לxנוס` עובר את הsuspicion check ומגיע לLLM בצורה גולמית. הLLM רואה "text מוזר" ומאשר.

**הפתרון:** להוסיף לisSuspicious: `if (/[\u05D0-\u05EA][a-zA-Z][\u05D0-\u05EA]/.test(text)) return true;`

### בעיה 3: פרומפט LLM לא מספיק מחמיר / מקיף

**הפרומפט הנוכחי** (לפני שינוי ה-llm.js שנעשה) מדבר על "swearing, offensive language, insults, death wishes" — **לא מזכיר:**
- תוכן מיני / איומים מיניים
- הפניות לאונס
- גידופי שואה
- גידופים עם Latin substitutions

### בעיה 4: מילים חסרות ב-CURSE_WORDS

חסרות לחלוטין:
- גידופים ערביים נפוצים: `דבע`, `חמאר`, `יכלב`
- `ניגר` (n-word בעברית)
- `לגזים` / הפניות לשואה כגידוף

### בעיה 5: False positives ממקור לא ידוע

כמה הודעות תמימות נחסמו מסיבה לא ברורה ("ניר אמר לי לעשות את זה", "היידה", "היי", "נדבר אחרי"). **אין לוג שמציין איזו מילה מ-CURSE_WORDS גרמה לחסימה** — בלתי אפשרי לאבחן.

## חלק ה — המלצות (לפי עדיפות)

### 🔴 עדיפות 1 — Critical

**א. הוסף גבולות מילה עברית לmatch של מילים קצרות (≤4 אותיות)**

בruleEngine.js, ב`smartMatchesForbidden`, עבור `normForbidden.length <= 4`:
```javascript
// במקום: normMessage.includes(normForbidden)
// השתמש ב: regex עם lookbehind/lookahead עברי
const hebrewBoundary = new RegExp(
    '(?<![\\u05D0-\\u05EA])' + normForbidden + '(?![\\u05D0-\\u05EA])'
);
if (hebrewBoundary.test(normMessage)) return true;
```
זה יפתור: `חריגה`, `תחתית`, `אנטיוכוס`, `ערבית`, `מנבל`, ועוד.

**ב. הוסף זיהוי Latin-בתוך-עברית לisSuspicious**

```javascript
// אות לטינית אחת בין עבריות — דפוס עקיפה
if (/[\u05D0-\u05EA][a-zA-Z][\u05D0-\u05EA]/.test(text)) return true;
```

**ג. הוסף ל-CURSE_WORDS**

```javascript
// גידופים ערביים
'דבע', 'דאבע',        // Arabic sexual slur (passed LLM 3 times)
'ניגר',               // n-word בעברית
'חמאר',               // ערבית: טיפש/חמור
'יכלב', 'יכאלב',     // ערבית: כלב כגידוף
// גידופי שואה
'לגזים', 'לתאי גז',  // הפניה לתאי גז
```

### 🟠 עדיפות 2 — High

**ד. שפר את פרומפט הLLM** (כבר בוצע חלקית ב-llm.js — לוודא שמכסה):
- תוכן מיני (גם "ספרותי" / עקיף)
- איומי אונס
- גידופי שואה (`לגזים`, `לתאי גז`)
- Latin substitutions בתוך עברית

**ה. הוסף debug logging ל-ruleEngine**

```javascript
// בcheckForbiddenMessages, כשמוצאים match:
if (process.env.DEBUG_RULES) {
    logger.debug(`Rule match: "${content}" → word: "${forbidden}"`);
}
```
בלי זה, impossible לאבחן false positives.

### 🟡 עדיפות 3 — Medium

**ו. בדוק ל-CONTEXT_WORDS להוסיף:**
```javascript
'נרתיק',   // vagina (medical) — used offensively
'בתולים',  // virginity — used in threats
'יבתול',   // deflower
'לגזים',   // if not in CURSE_WORDS directly
```

**ז. שקול להסיר `מרגל` מ-CURSE_WORDS** — "מרגל" (spy) הוא גידוף רך בהקשר פוליטי, גורם לפעמים לחסימה מוגזמת.

**ח. בדוק מקרה `Ben zona`** — לוודא ש-buildTransliterationPattern תופסת `ben zona` (שתי מילים).
